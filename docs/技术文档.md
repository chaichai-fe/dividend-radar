# 红利雷达 · 技术文档

> A 股红利类资产（红利 ETF / 高股息银行 / 能源高股息）的估值评级与组合管理 Web App。
>
> ⚠️ 本项目仅供学习研究，不构成任何投资建议。

---

## 1. 项目概述

「红利雷达（Dividend Radar）」是一个面向 A 股红利类资产的个人投资辅助工具，围绕三件事展开：

1. **行情汇总**：统一展示红利 ETF、高股息银行、能源高股息（煤炭 / 电力 / 石油）标的的实时净值 / 股价、当日涨跌幅与股息率。
2. **估值评级**：以「当前指标在多年历史区间中的分位」衡量低估 / 高估程度，给出五档评级与加仓建议。
3. **组合管理**：把标的加入个人组合，自动测算组合加权股息率、预计年 / 月被动收入与各标的占比，并按自然日记录历史快照绘制走势图。

这是一个**单用户、无需登录**的个人工具。持仓与快照数据存储在 Cloudflare D1，行情类数据实时抓取自公开接口并做边缘缓存。

## 4. 架构与数据流

### 4.1 整体架构

应用运行在 **Cloudflare Workers** 之上，由 TanStack Start 提供 SSR。前端与服务端逻辑同仓，服务端逻辑以 `createServerFn` 形式定义，前端通过 TanStack Query 调用。

```
浏览器 ── SSR/CSR ──► TanStack Start (Cloudflare Worker)
                          │
              ┌───────────┼─────────────────────────┐
              ▼           ▼                           ▼
        Server Fns   Cloudflare D1            外部公开行情接口
       (行情聚合)     (持仓 / 快照)      腾讯行情 / 韭圈儿 / 东方财富
              │                                       ▲
              └────── Cloudflare Cache API（边缘缓存）─┘
```

### 4.2 请求生命周期（以概览页为例）

1. 路由 `loader` 在 **SSR 阶段**用 `queryClient.ensureQueryData` 并行预取 `etfs`/`banks`/`cyclicals`/`holdings`，随后 dehydrate 到 HTML。
2. 客户端水合时，同样的 `queryKey` 直接命中已预取缓存，**首屏无骨架屏、省一次往返**。
3. 组件内用 `useQuery` 读取同一 `queryKey`，派生出「低估加仓榜」「股息率榜」「组合年被动收入」等展示数据。

### 4.3 数据缓存策略（两级）

**第一级 · TanStack Query（客户端，见 `root-provider.tsx`）**——「手动刷新」策略：

```ts
staleTime: Infinity, gcTime: Infinity,
refetchOnWindowFocus: false, refetchOnMount: false,
refetchOnReconnect: false, refetchInterval: false,
```

数据永不自动过期，页面切换 / 窗口聚焦 / 断网重连都不重取；仅**点击刷新按钮**（`query.refetch`）或**增删改持仓后手动失效**（`invalidateQueries`）才更新。目的：减少对上游接口的请求频次、行为可预期。

**第二级 · Cloudflare Cache API（边缘，见 `quotes.ts`）**——按数据变化频率分档：

| 数据类型 | 来源接口 | 缓存 TTL |
| --- | --- | --- |
| 实时净值 / 股价 / 涨跌幅 | 腾讯行情 `qt.gtimg.cn` | 45 秒 |
| 指数股息率 / PE / PB | 韭圈儿 `api.jiucaishuo.com` | 12 小时 |
| 每股现金分红（dps） | 东方财富 `datacenter-web.eastmoney.com` | 12 小时 |

所有抓取都有 `AbortSignal.timeout` 超时与 try/catch 兜底，**抓取失败时回退到 `lib/*-data.ts` 中的静态种子值**，保证页面始终可用。

## 5. 服务端函数（Server Functions）

均通过 `@tanstack/react-start` 的 `createServerFn` 定义，运行在 Worker 内，可直接访问 D1 与外部接口。

### 5.1 行情聚合

- **`listEtfs`**（`server/etfs.ts`）：并行抓取实时净值与实时指数股息率，与 `INDEXES` 静态估值区间合并成 `EtfQuote[]`。股息率优先用实时指数股息率（含当前价格因素），否则回退静态 `yieldNow`。返回附带 `liveCount` / `yieldLiveCount` 便于展示数据新鲜度。
- **`listBanks`**（`server/banks.ts`）：并行抓取实时股价与每股分红，动态计算 **股息率 = 每股分红 ÷ 现价 × 100**。
- **`listCyclicals`**（`server/cyclicals.ts`）：同银行口径，覆盖煤炭 / 电力 / 石油等能源高股息个股。

### 5.2 抓取工具

- **`quotes.ts`**：`fetchLiveQuotes(symbols, cacheName, ttl=45)`。symbols 形如 `sh510880,sz159581`，解析腾讯行情返回文本（`~` 分隔字段，取现价与涨跌幅）。内含 `readEdgeCache` / `writeEdgeCache` 供其它抓取模块复用。
- **`index-yields.ts`**：`fetchIndexYields(cacheName, ttl=12h)`，一次性拉全部指数股息率（口径：近 12 月现金分红 ÷ 指数当前点位），按归一化指数代码建映射。
- **`dividends.ts`**：`fetchDividends(codes, cacheName, ttl=12h)`，从东方财富分红明细按**前瞻口径**算 dps —— 取「最新已公告年度」（年报方案已发布，含预案）的中期 + 年末每股分红合计，及时反映最新分红调整。

### 5.3 持久化（D1）

- **`portfolio.ts`**：`listHoldings` / `addHolding`（存在则累加）/ `setHoldingAmount`（直接覆盖）/ `removeHolding`。写操作用 Drizzle 的 `onConflictDoUpdate` 实现 upsert。
- **`snapshots.ts`**：`listSnapshots` / `upsertSnapshot`（按 `day` 主键覆盖）/ `deleteSnapshot`。

## 7. 估值评级方法论（`lib/rating.ts`）

方法论核心是**分位（percentile）**：把当前指标放到该标的 / 指数的多年历史区间里，看它有多「便宜」。透明、可解释。

### 7.1 单因子（股息率）— `valuate`

```
percentile = (yieldNow - yieldLow) / (yieldHigh - yieldLow) × 100   // 0~100，越高越便宜
```

按分位映射到五档评级：

| 分位 | 等级 | 标签 | 星级 | 值得加仓 | tone |
| --- | --- | --- | --- | --- | --- |
| ≥ 70 | deep-undervalued | 显著低估 | ★★★★★ | 是 | buy-strong |
| ≥ 55 | undervalued | 低估 | ★★★★ | 是 | buy |
| ≥ 45 | fair | 合理 | ★★★ | 否 | hold |
| ≥ 30 | slightly-high | 略偏高 | ★★ | 否 | watch |
| < 30 | overvalued | 高估 | ★ | 否 | avoid |

### 7.2 多因子（ETF）— `valuateMulti`

在股息率之外叠加 PE、PB 两个「便宜度」分位，加权综合：

```
综合分位 = 0.5×股息率分位 + 0.3×PB分位 + 0.2×PE分位
```

- 股息率：**越高越便宜**（`pctHigherCheaper`）
- PE / PB：**越低越便宜**（`pctLowerCheaper`）
- 权重 `FACTOR_WEIGHTS = { yield: 0.5, pb: 0.3, pe: 0.2 }`——红利指数多为金融 / 周期 / 重资产，PB 比 PE 更具参考意义，故 PB 权重高于 PE。

综合分位复用 `valuate` 的五档分档逻辑输出评级，并附各因子分位明细（`factors`）。

### 7.3 银行 / 能源股评级

不走历史区间分位，而是按**当前股息率绝对值**分档（`rateBankYield` / `rateCyclicalYield`，见对应 `*-data.ts`），因其股息率由实时股价动态测算。概览页中「股息率 ≥ 5%」记为高股息标的。

### 7.4 股债利差

```
股债利差 = 股息率 − BOND_YIELD_10Y(1.78%)   // equityBondSpread()
```

用于直观展示相对债券的配置价值，利差越大越有吸引力。

## 8. 前端与 UI

### 8.1 路由与页面

文件式路由（`src/routes/`），`routeTree.gen.ts` 由 `tsr generate` 自动生成。

| 路由 | 页面 | 说明 |
| --- | --- | --- |
| `/` | 概览 | 组合年被动收入卡片、各类平均股息率、低估加仓榜、银行 / 能源股息率榜 |
| `/etfs` | 红利 ETF 估值 | 三因子评级榜、加仓建议 |
| `/banks` | 高股息银行 | 按实时股价测算股息率 |
| `/cyclicals` | 能源高股息 | 煤炭 / 电力 / 石油 |
| `/portfolio` | 我的组合 | 搜索添加、组合测算、历史走势 |

### 8.2 根布局（`__root.tsx`）关键点

- **首屏 loading 遮罩**：关键 CSS 内联在 `<head>`（`criticalCss`），随 HTML 立即渲染，应用挂载后加 `app-ready` 类淡出；带 12s failsafe 动画兜底，避免 PWA 冷启动白屏。
- **主题防闪烁**：`themeInitScript` 在 `<head>` 内以 `dangerouslySetInnerHTML` 同步执行，首帧即确定明暗色。
- **PWA**：`ServiceWorkerRegistrar` 仅在生产环境注册 `/sw.js`；`<head>` 引入 `manifest.json` 与各尺寸图标。
- **响应式导航**：桌面顶部横排 + 移动端第二行导航。
- **DevTools**：集成 TanStack Router / Query DevTools（`@tanstack/react-devtools`）。

### 8.3 组合测算逻辑（概览 & 组合页）

把 ETF / 银行 / 能源股统一为 `code → { price, dividendYield }` 映射，组合内不区分类别：

```
某标的市值 = 持有数量 × 现价
组合总市值 = Σ 各标的市值
预计年收入 = Σ (市值 × 股息率/100)
组合股息率 = 年收入 / 总市值 × 100
月均收入   = 年收入 / 12
```

组合非空时，组合页每天首访会自动 `upsertSnapshot` 记录当日快照，`TrendChart`（零依赖 SVG）据此绘制走势。

### 8.4 设计与组件

- shadcn/ui 组件位于 `components/ui/`（badge / button / card / dialog / input / label / select / skeleton / sonner / table）。
- 设计变量与自定义类（`island-*`、`feature-card`、`display-title`、`--lagoon-deep` 等）定义在 `styles.css`。
- Toast 使用 `sonner`（`richColors`，顶部居中）。

## 11. 数据来源与免责声明

- **净值 / 股价 / 涨跌幅**：实时抓取腾讯行情（`qt.gtimg.cn`），失败回退静态种子。
- **指数股息率 / PE / PB**：韭圈儿指数估值 + `etf-data.ts` 静态区间（东方财富、红利查等参考值，变化缓慢）。
- **每股分红**：东方财富「分红送配」数据中心，前瞻口径（含预案）。

> ⚠️ 所有数据均来自公开网络，股息率与估值区间为参考值，可能存在滞后或误差。**本项目仅供学习与研究，不构成任何投资建议。**

## 12. 扩展指南

- **新增一只 ETF / 指数**：在 `lib/etf-data.ts` 的 `ETFS` / `INDEXES` 中补充条目（含估值区间与种子值），无需改动服务端逻辑。
- **新增银行 / 能源股**：在 `lib/bank-data.ts` / `lib/cyclical-data.ts` 补充；股息率会自动按实时股价 + 抓取分红计算。
- **调整评级口径**：修改 `lib/rating.ts` 的分档阈值或 `FACTOR_WEIGHTS` 权重。
- **调整缓存新鲜度**：修改各抓取函数的 `ttl` 参数（`quotes.ts` / `index-yields.ts` / `dividends.ts`）。
- **新增页面**：在 `src/routes/` 新建文件后运行 `bun run generate-routes`（或依赖 dev 自动生成）。
```
